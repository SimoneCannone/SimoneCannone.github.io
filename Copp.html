About the Project

Copperfield Engine is a 3D game engine created by just two people. It aims to offer the essential features for game development, with an easy-to-use API that follows a data-driven design with the use of ECS.

The only libraries used are glfw, glew, imgui, tinyobjloader, StbImage and OpenAL. We have tried to keep them to a minimum, having our own math and collision library, as well as physics.

Main Tasks

In this project, my main responsibilities included the implementation of an ECS (Entity Component System) alongside my teammate. This system was designed to provide a modular and flexible architecture, allowing the creation of entities composed of various components. These components could either be custom-defined by the user or built-in elements of the engine, ensuring a scalable and adaptable system for game development. The ECS was structured to optimize performance while maintaining ease of use, enabling efficient entity management and interaction within the engine.

Additionally, I worked on the development of a multitasking system to enhance asset loading efficiency. By implementing multithreading, we were able to load multiple assets simultaneously, significantly reducing loading times and improving overall performance. This system allowed the engine to handle asset streaming dynamically, ensuring a smoother experience for both developers and end users.

My primary focus within the project was on the rendering pipeline, specifically deferred rendering and post-processing effects. I designed and optimized the rendering system to deliver high-quality visuals while maintaining performance efficiency. The post-processing pipeline included various graphical enhancements such as bloom, depth of field, and color correction, all of which contributed to an immersive visual experience.

Furthermore, I was responsible for implementing the engine’s sound system using the OpenAL library. This involved integrating spatial audio, handling sound effects and background music, and ensuring smooth audio playback with minimal performance impact. The system was designed to be flexible and customizable, allowing developers to easily manage and manipulate audio within the engine.

Overall, my contributions focused on optimizing both the visual and auditory aspects of the engine while ensuring smooth performance and scalability.

Main Features

Entity Component System

Implemented a custom Entity Component System (ECS) in a proprietary engine using C++, allowing dynamic management of game entities and components for greater flexibility and scalability in gameplay development.

Multi-threaded Asset Load

Developed a multi-threaded asset loading system in the custom engine, significantly reducing loading times and improving performance by asynchronously handling resources on multiple threads.

Forward and Deferred Rendering

Created custom forward and deferred rendering pipelines using OpenGL in a proprietary engine, optimizing real-time lighting and shading for stunning graphics and efficient performance.

Action-based Input

Built a flexible action-based input system in C++, offering responsive control mechanics and smooth gameplay across platforms in the custom engine, including mobile and desktop.

Shadow Mapping

Implemented custom shadow mapping techniques in OpenGL within the proprietary engine, enhancing the depth and realism of lighting and shadows in the game world.

Physics and Collisions

Developed a custom physics and collision system in C++ for the proprietary engine, ensuring accurate and realistic interactions between game objects and enhancing the overall gameplay experience.

Sound and Music

Implemented a custom sound system in the proprietary engine using OpenAL, enabling dynamic sound effects, spatial audio, and immersive music playback, enhancing the auditory experience for players.
