About the Project

Multi-agent system developed in Unity where different types of AI-controlled units compete against each other.

The system is designed to simulate autonomous decision-making, where each AI-controlled unit follows specific behaviors and adapts to dynamic conditions. These agents interact in real time, creating complex and emergent gameplay scenarios.

By utilizing Unity's AI tools and custom logic, the system enables competitive and cooperative strategies among agents. The result is a dynamic environment where AI behaviors evolve, making each session unpredictable and engaging.

Main Tasks

The three unit types are:

Explorer: Equipped with a node detection system that allows it to navigate the map based on previously visited areas. Once it finds something of interest, it alerts the Leader.

Miner: Responsible for gathering resources necessary for unit creation, ensuring a steady supply for expanding the unit population.

Warrior: Specializes in combat, featuring its own vision range, attack capabilities, and unique stats. It engages enemies based on strategic positioning and the Leader’s commands.

Each unit has an internal finite state machine (FSM) that determines its current action and allows for a degree of autonomous decision-making. However, primary orders are issued by the Leader, which uses a message and bidding system to assign tasks efficiently. The bidding mechanism prioritizes units based on their distance to the target, ensuring that the most suitable unit executes the task in the shortest time possible.

This system enables dynamic and strategic interactions between AI-controlled units, optimizing resource management, exploration, and combat effectiveness.

Main Features

State Machine

Each agent has its own state machine that allows it to perform actions individually without the need for control.

Message System

Communication system between leaders and agents that allows the leader to deliver tasks to the agents through messages that are filtered by agent bids, thus determining which agent receives the task.

Map Discovery

Exploration system that allows agents to move based on what explorers have already discovered, since the map is composed of nodes that are marked as visited or to be visited.

Decisions

Decisions are made based on the agents' characteristics, since in our case, the task will always be assigned to the closest agent with the bidding system, but bidding could be based on life, strength, or anything else you specify.
